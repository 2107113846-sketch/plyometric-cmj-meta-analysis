# 数据提取教程：从 PDF 论文到 Meta 分析数据表

## 概述

你现在有 **91 篇论文的 PDF 全文**，需要从每篇论文中提取关键统计数据，填入 `data_extraction_master.csv`。

这是一项需要耐心但**极其重要**的工作——**数据提取错误 = Meta 分析结果错误**。

---

## 第一步：理解你要提取什么

### 核心数据（每篇论文必须提取）

| 数据项 | 说明 | 在论文中的位置 |
|--------|------|---------------|
| **样本量 (n)** | 干预组和对照组各多少人 | Methods → Participants |
| **CMJ 前测均值** | 干预开始前 CMJ 高度的平均值 | Results → Table/Figure |
| **CMJ 前测标准差 (SD)** | 干预开始前 CMJ 高度 SD | Results → Table/Figure |
| **CMJ 后测均值** | 干预结束后 CMJ 高度平均值 | Results → Table/Figure |
| **CMJ 后测标准差 (SD)** | 干预结束后 CMJ 高度 SD | Results → Table/Figure |
| **干预周期** | 训练持续了多少周 | Methods → Procedures |
| **每周频率** | 每周训练几次 | Methods → Procedures |

### 次要数据（如有则提取）

- Squat Jump (SJ) 高度
- Drop Jump (DJ) 高度
- 短跑时间 (10m/20m/30m)
- 灵敏性测试 (T-test / Illinois / 505)
- 参与者年龄、性别、运动项目

## 第二步：打开 PDF，按顺序阅读

### 2.1 论文结构速查

每篇运动科学论文基本遵循固定结构：

```
Title & Abstract        → 确认是你要的论文
Introduction            → 跳过，不需要提取数据
Methods                 → 样本量、干预参数 ⭐
  ├─ Participants       → n、年龄、性别、运动项目
  ├─ Procedures         → 训练方案（周期、频率、类型）
  └─ Statistical Analysis → 确认用的什么检验方法
Results                 → CMJ Mean±SD 在这里！ ⭐⭐⭐
  ├─ 文字描述           → 有时夹着数据
  ├─ Table 1            → 通常是参与者基线特征
  ├─ Table 2            → 通常是主要结局（CMJ/SJ/DJ）
  └─ Figures            → 柱状图/折线图（数据在上面）
Discussion              → 跳过，不需要提取
```

### 2.2 实战示例：以 `chaabene2017.pdf` 为例

这是一篇典型的 plyometric training 论文。我们来走一遍完整流程。

#### Step 1: 找样本量

打开 PDF，翻到 **Methods → Participants** 部分（通常在 2-4 页）。

你会看到类似这样的文字：

> "Twenty-five prepubertal male soccer players participated in this study and were randomly assigned to a plyometric training group (PTG; n = 13) and a control group (CG; n = 12)."

从中提取：
- `n_total` = **25**
- `n_int` = **13**
- `n_ctrl` = **12**

> 💡 **提示**：有时论文会在 Table 1 中重复给出样本量，可以和正文交叉核对。

#### Step 2: 找干预参数

在 **Methods → Procedures** 部分找训练方案描述：

> "The PTG participated in an 8-week plyometric training program... performed twice per week..."

提取：
- `int_duration_wk` = **8**
- `int_freq_pw` = **2**

#### Step 3: 找 CMJ 数据 ⭐ 最关键！

翻到 **Results** 部分。寻找包含 CMJ 数据的表格。通常标记为 **Table 2** 或类似。

典型的结果表格长这样：

```
Table 2. Performance results before and after training.

Variable        Group       Pre-test        Post-test
-----------------------------------------------------
CMJ (cm)        PTG         23.66 ± 2.83    26.85 ± 2.78*
                CG          23.41 ± 2.91    23.55 ± 2.87
-----------------------------------------------------
SJ (cm)         PTG         21.29 ± 3.04    24.12 ± 2.95*
                CG          21.05 ± 2.88    21.33 ± 2.76
```

从表格中提取：

**CMJ 数据：**

| 列名 | 值 | 含义 |
|------|-----|------|
| `cmj_pre_mean_int` | **23.66** | PTG 前测 CMJ 均值 |
| `cmj_pre_sd_int` | **2.83** | PTG 前测 CMJ 标准差 |
| `cmj_post_mean_int` | **26.85** | PTG 后测 CMJ 均值 |
| `cmj_post_sd_int` | **2.78** | PTG 后测 CMJ 标准差 |
| `cmj_pre_mean_ctrl` | **23.41** | CG 前测 CMJ 均值 |
| `cmj_pre_sd_ctrl` | **2.91** | CG 前测 CMJ 标准差 |
| `cmj_post_mean_ctrl` | **23.55** | CG 后测 CMJ 均值 |
| `cmj_post_sd_ctrl` | **2.87** | CG 后测 CMJ 标准差 |

> ⚠️ **最常见的错误**：把标准差 (SD) 和标准误 (SEM) 搞混！
> - **SD** (Standard Deviation) = 数据的离散程度，是我们需要的
> - **SEM** (Standard Error of Mean) = 均值的精确度，**不是你需要的**
> - 如果论文只报告了 SEM，需要换算：**SD = SEM × √n**
> - 在表中看到 "±" 后面跟的数字，要去表注确认是 SD 还是 SEM

## 第三步：识别不同格式的数据报告方式

运动科学论文有几种常见的报告数值的方式。

### 格式 A：表格（最常见，约占 70%）

如上例所示，数据在结果表格中。这是**最可靠**的来源。

**提取步骤：**
1. 找到 Table 标题包含 "performance"、"jump"、"CMJ" 的表格
2. 找到 CMJ 那一行
3. 横向读出 Pre 和 Post 列的值
4. 区分 PTG (干预组) 和 CG (对照组) 的行

### 格式 B：正文文字（约占 20%）

有时数据不在表格中，而是写在结果文字里：

> "Countermovement jump height increased significantly in the intervention group from 30.2 ± 4.5 cm at baseline to 35.8 ± 3.9 cm after training (p < 0.01), while the control group showed no significant change (31.1 ± 4.2 vs. 31.5 ± 4.0 cm, p > 0.05)."

提取：
- INT pre: **30.2 ± 4.5**
- INT post: **35.8 ± 3.9**
- CTRL pre: **31.1 ± 4.2**
- CTRL post: **31.5 ± 4.0**

### 格式 C：图表/Figure（约占 10%）

数据只在柱状图中。**无法精确提取数字。**

> ⚠️ 如果数据**只**在 Figure 中、正文和表格都没有给出精确数值：
> 1. 在 `extraction_notes` 中标记 "Data only in figures — needs digitization"
> 2. 考虑使用 WebPlotDigitizer 等工具从图中提取数据
> 3. 或者在 `extraction_status` 中标记为 `flagged`

## 第四步：填写 CSV 提取表

### 4.1 用 Excel 打开提取表

1. 打开 Excel
2. 文件 → 打开 → 浏览 → 选择 `d:\桌面\科研训练\data_extraction_master.csv`
3. 确保编码选 **UTF-8**

### 4.2 填写流程（每篇论文约 3-5 分钟）

```
1. Ctrl+F 搜索第一作者名字，定位到该论文的行
2. 打开对应的 PDF
3. 翻到 Methods，填入：
   - n_int, n_ctrl, n_total
   - sex, age_mean, age_sd
   - int_duration_wk, int_freq_pw, plyo_type
4. 翻到 Results，找到 CMJ 表格/数据，填入：
   - cmj_reported = "yes"
   - cmj_pre_mean_int, cmj_pre_sd_int
   - cmj_post_mean_int, cmj_post_sd_int
   - cmj_pre_mean_ctrl, cmj_pre_sd_ctrl
   - cmj_post_mean_ctrl, cmj_post_sd_ctrl
   - cmj_equipment（在 Methods 中找测量设备）
5. 如果有 SJ/DJ/Sprint 数据，同样填入
6. 检查数据是否有异常（见第五步）
7. extraction_status → "complete"
```

## 第五步：数据质量自检 🔍

填完每篇后，做一个快速检查：

### 检查清单

- [ ] CMJ 值是否在合理范围？（**15-80 cm** 是正常的）
  - 如果 ＜ 10 cm：可能单位是米，需要 ×100
  - 如果 ＞ 100 cm：可能是功率 (W) 而非高度 (cm)
- [ ] SD 是否合理？（通常是均值的 **5%-30%**）
  - 如果 SD ＞ 均值：可能填错了，或数据报告方式不同
- [ ] 样本量加起来等于总数？
  - n_int + n_ctrl ≈ n_total（可能有少量脱落）
- [ ] 前后测的 n 是否一致？
  - 如果后测 n 比前测少，说明有脱落，需要在 notes 中记录
- [ ] 干预组和对照组的前测值**大致接近**？
  - 如果前测差距很大（如 25 vs 35 cm），说明随机化可能有问题
- [ ] 干预组后测 **≥** 前测？（训练通常提高 CMJ）
  - 如果后测反而更低，可能是特殊情况，需要仔细确认
- [ ] 有无标记 * 或 † 表示显著性？
  - 不影响数值提取，但可以在 notes 中注明显著性

### 常见陷阱

| 陷阱 | 说明 | 如何避免 |
|------|------|----------|
| **SD vs SEM** | 论文可能报告标准误而非标准差 | 读表注 (Table note)，找 "Values are mean ± SD" 或 "mean ± SEM" |
| **单位混淆** | CMJ 可能是 cm、m 或 inches | 看表头或论文方法，确保统一成 **cm** |
| **多组设计** | 论文有 3+ 组（如高低量训练 + 对照） | 只提取 plyometric 组 vs 对照组，另一组记在 notes 中 |
| **多个时间点** | 论文有中期测试（如第4周） | 只提取最后一次测试（干预结束时）的数据 |
| **百分比变化** | 论文只报告 Δ% 而非绝对值 | 标记为 `flagged`，需要联系作者获取原始数据 |
| **合并报告** | 男女混合报告、不分组 | 如果可以拆分就拆分，否则标记 merged |
| **年龄/身高误认为 CMJ** | Table 1 中的身高数据 ≠ CMJ 高度 | ⚠️ 这是最常见的错误！确认你在看的是 performance/jump 表格，不是 demographics |

## 第六步：特殊情况处理

### 情况 1：论文没有 CMJ

有些论文研究的是 plyometric training，但结局指标可能不是 CMJ，而是：
- Drop Jump (DJ)
- Squat Jump (SJ)
- Sprint performance
- Reactive Strength Index (RSI)

如果完全没有 CMJ 数据：
- `cmj_reported` → `"no"`
- 在 `other_outcomes` 中记录实际报告的指标
- `extraction_notes` → "No CMJ; outcomes: DJ, sprint 20m"

### 情况 2：数据只在图表中

如果数值只显示在柱状图/折线图上，表格和正文都没有精确数字：
- `extraction_status` → `"flagged"`
- `data_source` → `"figure"`
- `extraction_notes` → "CMJ data only in Figure 2 — needs WebPlotDigitizer"

### 情况 3：论文不是 RCT

如果论文是横断面研究（比较不同人群）而非训练干预：
- 不需要提取 pre-post 数据
- `extraction_status` → `"excluded_posthoc"`
- 在 notes 中说明为什么排除

### 情况 4：重复论文

如果发现两篇论文报告了同一批数据：
- 保留发表较早或数据更完整的那篇
- 在另一篇的 notes 中标记 `"duplicate of [PMID]"`

## 第七步：数据记录约定

### 数值精度
- 保留和原文一样的精度（通常是 1-2 位小数）
- 不要自己四舍五入

### 缺失值
- 留空，不要填 "0" 或 "N/A"
- R 脚本会自动处理空值

### 单位统一
- CMJ/SJ/DJ 高度：统一为 **cm**
- 短跑时间：统一为 **秒 (s)**
- 如果原文用 m，换算 ×100
- 如果原文用 inches，换算 ×2.54

### 多组设计
如果论文有 3 组（如：高强度 plyometric、低强度 plyometric、对照组）：
- 选择**和对照组最可比的那组**作为主要提取对象
- 在 notes 中记录 "Also has [X] group: mean=XX SD=XX"

## 效率建议

### 每天做 10-15 篇

- 每篇约 3-5 分钟
- 每天 10-15 篇，约 1-1.5 小时
- 91 篇约需 **7-10 天**完成
- 不要一天做太多——疲劳会导致错误

### 遇到问题怎么办

1. 先看 DATA_DICTIONARY.md 确认字段定义
2. 看这篇教程的对应章节
3. 在 extraction_notes 中记录疑问
4. 标记 `extraction_status` 为 `flagged`
5. 继续下一篇，之后再回头处理 flagged 的

### 双人核对（如果有条件）

最好让另一个人独立提取 20% 的论文，然后对比：
- 如果差异 > 5%，重新检查
- 这是 Cochrane 推荐的质量控制方法
